# Reflection이란?

```java
@ExceptionHandler(Exception.class)
```

Java로 개발하다보면 위와 같이 `.class`라는 표현을 많이 작성하게 된다. 이처럼 `ClassName.class`의 표현은 실제로 개발자가 작성한 Java 클래스의 `Class` 객체에 대한 참조를 가져오는 것이다. (객체는 JVM의 클래스 로더를 통해 메모리 힙에 저장된다.)

`Class` 객체란 해당 클래스의 메타데이터와 구조를 포함하며, JVM 내에서 해당 클래스에 대한 정보를 나타낸다.

```java
Class<App> appClass = App.class;
```

따라서 위와 같이 `appClass`에 접근하면 `App.class`에 저장된 메타데이터를 이용하여 메소드, 필드, 생성자, 어노테이션 등 다양한 정보를 동적으로 조작할 수 있다.

이처럼 메타데이터에 대한 정보를 런타임에 조작하는 기능을 제공하는 것을 `Reflection`이라고 한다.

런타임에 정보를 조작하기 때문에 잘못 사용하는 경우 런타임 에러를 일으킬 수 있으며, 성능면에서 객체에 직접 접근하여 활용하는 것보다 좋지 않기 때문에 성능 이슈를 고려해야 한다.
